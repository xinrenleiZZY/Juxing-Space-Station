## 一、数据集概述

- **数据集名称：** 京津冀PM2.5空气质量多维度特征数据集(2015-2026)

## 四、字段关联关系

1. **主关键字段**：date + city (日期+城市) 组合唯一标识一条记录
2. **时间关联**：year、month、day、season等字段均从date字段派生
3. **事件关联**：pollution_event_id与event_duration、peak_intensity等字段一一对应
4. **空间关联**：regional_avg_pm25、deviation_from_regional_avg等字段基于city字段计算

## 五、数据质量保证

1. **数据完整性**：所有32个字段结构完整，无缺失字段
2. **数据一致性**：时间、空间等维度的数据逻辑一致
3. **数据准确性**：PM2.5浓度值符合GB 3095-2012标准范围
4. **数据有效性**：所有计算字段均基于科学算法生成

## 六、数据处理流程说明

1. **原始数据采集**：从公开环境监测网站采集基础PM2.5数据
2. **数据预处理**：城市名称标准化、缺失值处理、异常值检测
3. **特征工程**：生成32个多维度特征（时间、事件、空间、政策、健康风险）
4. **质量验证**：数据完整性检查、逻辑一致性验证
5. **数据整合**：构建最终的多维度特征数据集
